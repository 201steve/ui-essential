# 1. div 태그에 hidden 속성
* hidden 속성을 boolean으로 주면 요소를 화면에서 숨기고 공간도 차지하지않게 만든다.
* CSS의 display:none;과 동일하게 스크린 리더 드에서도 접근할 수 없도록 완전히 숨긴다.
* 이 속성은 자바스크립트로 hidden 속성을 제거해서 요소를 다시 보이게할 수 있다.

# 2. new Set()은 중복되지 않는 고유값을 다뤄야 할 때 유용하다.
* 동일한 값을 두번 저장하지않는다.
* new Set(배열)을 감사면 즉시 중복된 요소가 제거된다.
* 데이터 유일성을 보장해야 할 때 유영하고, add()로 요소를 추가할 때 이미 존재하는 값은 무시된다.
* Set 내의 요소는 삽입된 순서대로 저장되고, 반복시 순서가 유지된다.
* has() 메서드로 특정 값으 존재 여부를 배열보다 빠르게 확인할 수 있다.

# 3. ARIA
원칙 1

role은 이름표가 아니라, 그 역할에 맞는 키보드/상태/동작까지 제공하겠다는 약속이다.

원칙 2

ARIA는 기존 HTML 의미를 더 좋게 보강할 수도 있지만, 잘못 쓰면 원래 의미를 덮어써서 접근성을 망칠 수도 있다.

전체 결론

가능하면 네이티브 HTML을 먼저 쓰고, ARIA는 정말 필요한 의미와 상태만 신중하게 보강하는 용도로 써야 한다.

## 3-1 aria-hidden
이 속성이 있으면 접근성 트리에서 제외된다.즉, 시각 사용자는 아이콘을 볼 수 있는데
스크린 리더 사용자는 이 span이 없는 것처럼 처리된다. 

단순히 화면에서 안보이게 만들거나, 레이아웃에서 제거하는게 아니라
보조기술이 이 요소를 인식하지 않게 하는것.



왜 이렇게 사용하냐면,
```
<button>
배송정보
<span>▼</span>
</button>
```
이렇게 되면 스크린리더 사용자는 span 정보까지 듣게 된다. span은 단순 장식용이지 기능을 하는게 아니기때문에
환경에 따라 불필요한 정보가 섞일 수 있어서 aria-hidden으로 접근성 트리에서 제외하는게 맞다.
## 3-2 aria-expended
이 속성이 있으면 보조기술 사용자는 열려있는지 닫혀있는지 상태를 듣게된다.
